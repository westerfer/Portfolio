Development Mantra

Keep. It. Simple. Stupid.

K.I.S.S. — This is one of most important fundamental rules that I like to follow when developing an application or creating an interface. What I like about this rule is that there's a point in every project when this rule needs to be deconstructed.
 
Visual Design

I like to approach a project design with a simple yet minimalistic process. I would rather have the user focus on the business side of the project rather than the creativeness.

 
User Research

Without the guidance of how an individual uses a program, you're essentially navigating without a compass.

 
Code Development

From the projects build from beginning to end, writing code with future project extensions in mind will enhance future development.
